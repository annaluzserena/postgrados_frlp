# Changelog — Wireframes & Prototipado UX/UI

Registro de cambios del proyecto **[Fénix Posgrado - Grupo 01]**.
Este changelog documenta la evolución de wireframes, prototipos y decisiones de diseño UX/UI.

El formato sigue las convenciones de [Keep a Changelog](https://keepachangelog.com/es-ES/1.0.0/) adaptadas a procesos de diseño, y versionado [Semántico](https://semver.org/lang/es/) (MAJOR.MINOR.PATCH).

---

## Convenciones

- **Added** — Nuevas pantallas, flujos o componentes agregados.
- **Changed** — Modificaciones a wireframes/prototipos existentes.
- **Fixed** — Correcciones de inconsistencias, errores de usabilidad o de diseño.
- **Deprecated** — Pantallas/flujos marcados para reemplazo.
- **Removed** — Pantallas, flujos o componentes eliminados.
- **Research** — Hallazgos de research/testing que motivaron cambios.

Cada entrada debe referenciar, cuando aplique: herramienta de diseño (Figma/Adobe XD/etc.), link al artefacto, y responsable.

## Notas de versionado sugerido

| Tipo de cambio | Ejemplo | Incremento |
|---|---|---|
| Cambio mayor de flujo o arquitectura de información | Rediseño completo de un módulo | MAJOR |
| Nueva pantalla, flujo o componente | Agregar pantalla de checkout | MINOR |
| Ajuste menor, fix de usabilidad | Corregir alineación, copy, estado de error | PATCH |
